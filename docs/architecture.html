SDK Layers

MeroJs is structured as a set of composable clients, all built on a single HttpClient abstraction. You create a MeroJs instance which wires everything together.

src/mero-js.ts
Main MeroJs class. Creates and holds all sub-clients. Entry point for all operations.
src/http-client/
HttpClient — thin wrapper around fetch. Handles base URL, auth headers, timeout, and signal combination.
src/auth-api/
AuthApiClient — login, logout, refresh token. Writes tokens to TokenStore.
src/auth/
Auth helpers: parseAuthCallback() reads tokens from URL hash. buildAuthLoginUrl() constructs the auth redirect URL.
src/admin-api/
AdminApiClient — manage contexts, applications, identities, and node resources.
src/rpc/
RpcClient — execute WASM application methods (query/mutate) on a context.
src/events/sse.ts
SseClient — Server-Sent Events subscription for real-time node events.
src/events/ws.ts
WsClient — WebSocket subscription for real-time node events.
src/token-store/
TokenStore interface and LocalStorageTokenStore implementation. Pluggable token persistence.

Error Model

The SDK throws on errors. There are two error categories:

HTTPError

Thrown when response.ok === false (status 4xx, 5xx). Contains status, statusText, and the response body.

try {
  await client.execute(...);
} catch (err) {
  if (err instanceof HTTPError) {
    // err.status, err.body
  }
}

Network / Abort Errors

Network failures, DNS errors, and AbortError (user cancellation or timeout) are re-thrown as-is. The SDK distinguishes user aborts from timeout aborts for intelligent retry logic.

if (err.name === 'AbortError') {
  // Check if timeout or user cancel
  if (err.isTimeout) { /* retry */ }
}

Signal Combination & Cancellation

mero-js implements signal combination — you can pass your own AbortSignal and the SDK internally merges it with a timeout signal:

// The SDK creates its own timeout signal internally
// and combines it with your signal:

const controller = new AbortController();
const result = await client.get('/api/data', {
  signal: controller.signal, // user cancellation
  timeoutMs: 5000, // timeout signal (merged)
});

// Abort from your side at any time:
controller.abort();

This ensures requests always clean up properly — either from a user-initiated cancel or from a timeout — without leaving dangling fetch calls.

Dependency Injection

No global singletons. Create one MeroJs instance per application context:

import { MeroJs } from '@calimero-network/mero-js';

const sdk = new MeroJs({
  baseUrl: 'https://node.example.com',
  timeoutMs: 10_000,
  tokenStore: myCustomStore, // optional
});

// All sub-clients share the same HttpClient instance
sdk.auth // AuthApiClient
sdk.admin // AdminApiClient
sdk.rpc // RpcClient
