Error Model

HTTPError vs network errors — what throws and how to handle each

HTTPError

Thrown when the server responds with a non-OK status (response.ok === false, i.e. 4xx or 5xx):

class HTTPError extends Error {
  status: number;      // HTTP status code (400, 404, 500, …)
  statusText: string; // HTTP status text
  body: unknown;     // Parsed response body (JSON or text)
}
import { HTTPError } from '@calimero-network/mero-js';

try {
  await sdk.admin.getContext('ctx_abc');
} catch (err) {
  if (err instanceof HTTPError) {
    if (err.status === 404) {
      console.log('Context not found');
    } else if (err.status === 401) {
      // Token expired — refresh and retry
      await sdk.auth.refresh();
    }
  }
}

Network & Abort Errors

Network failures and aborts are re-thrown as-is:

TypeError (fetch failed)
DNS failure, refused connection, or other network-level error. Check err instanceof TypeError.
AbortError (user cancel)
The user's AbortSignal fired. err.name === 'AbortError' and err.isTimeout === false.
AbortError (timeout)
The SDK's internal timeout signal fired. err.name === 'AbortError' and err.isTimeout === true. Safe to retry.

Full Error Handler Pattern

import { HTTPError } from '@calimero-network/mero-js';

async function withRetry(fn: () => Promise<unknown>) {
  try {
    return await fn();
  } catch (err) {
    if (err instanceof HTTPError) {
      if (err.status === 401) {
        await sdk.auth.refresh(); // refresh token
        return await fn(); // retry once
      }
      throw err; // other HTTP errors: rethrow
    } else if (err instanceof Error && err.name === 'AbortError') {
      if ((err as any).isTimeout) {
        return await fn(); // timeout: safe to retry
      }
      // user abort: don't retry
    } else {
      throw err; // network error: rethrow
    }
  }
}
