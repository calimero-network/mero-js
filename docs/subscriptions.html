Subscriptions

Real-time events from the node via Server-Sent Events or WebSocket

SseClient — Recommended

Access via sdk.events. Uses the browser native EventSource API to receive a continuous stream of events from the node.

import { MeroJs } from '@calimero-network/mero-js';

const sdk = new MeroJs({ baseUrl: 'http://localhost:2428' });
await sdk.authenticate({ username: 'admin', password: 'pass' });

// Subscribe to events in a context
const unsubscribe = sdk.events.subscribe(['context-id-1'], (event) => {
  console.log(event.contextId, event.executorPublicKey, event.data);
});

// Stop listening
unsubscribe();

// Or close all connections
sdk.close();

SseEventData

interface SseEventData {
  contextId: string;                // context the event came from
  executorPublicKey: string;         // identity that executed the call
  outcome: Record<string, unknown>; // execution result
  data?: unknown;                  // event payload from WASM
}

WsClient — Experimental

Access via sdk.ws. Provides the same interface as SseClient over WebSocket. Logs a console warning on first use — prefer SSE in production.

// Same interface as SseClient:
const unsubscribe = sdk.ws.subscribe(['ctx-id'], (event) => {
  console.log(event);
});

⚠ WsClient is experimental. The API may change in a future release.

Multi-context Subscriptions

Pass multiple context IDs to subscribe to all of them with a single connection:

const unsub = sdk.events.subscribe(
  ['ctx-1', 'ctx-2', 'ctx-3'],
  (event) => {
    // event.contextId tells you which context fired
    console.log(`Event from ${event.contextId}`);
  }
);

React Usage

If you're using React, use the useSubscription hook from @calimero-network/mero-react instead — it handles lifecycle automatically:

import { useSubscription } from '@calimero-network/mero-react';

useSubscription([contextId], (event) => {
  // auto-unsubscribes on unmount
  // re-subscribes if contextIds change
  handleEvent(event);
});
