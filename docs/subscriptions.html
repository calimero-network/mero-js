Subscriptions

Real-time event streaming via SSE and WebSocket

SSE Client — sdk.sse

Server-Sent Events — one-directional, HTTP-based event stream. Works through firewalls and proxies that may block WebSockets:

// Subscribe to node events over SSE
const sse = sdk.sse;

const unsub = sse.subscribe({
  contextId: 'ctx_...',
  onEvent: (event) => {
    console.log('Event:', event);
  },
  onError: (err) => {
    console.error('SSE error:', err);
  },
});

// Stop listening:
unsub();

WebSocket Client — sdk.ws

Full-duplex WebSocket connection. Lower latency for high-frequency events:

const ws = sdk.ws;

const unsub = ws.subscribe({
  contextId: 'ctx_...',
  onEvent: (event) => {
    console.log('WS event:', event);
  },
});

// Stop and close connection:
unsub();

Event Shape

interface NodeEvent {
  contextId: string;
  eventType: string;  // e.g. 'StateMutation', 'ContextCreated', …
  payload: unknown;  // event-specific data (JSON)
  timestamp: number; // Unix timestamp ms
}

Choosing SSE vs WebSocket

Use SSE when

  • Firewall or proxy environment
  • Read-only event stream (no need to send)
  • Simpler reconnect behaviour needed

Use WebSocket when

  • Low-latency is critical
  • High-frequency events (>10/sec)
  • Bidirectional communication in future
